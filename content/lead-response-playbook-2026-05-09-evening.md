# Step3D — внутренний playbook обработки заявок

Дата: 2026-05-09, вечерний спринт.

Цель: быстро превращать входящие заявки с сайта/Telegram в понятный расчёт без лишней переписки.

## 1. Классификация заявки за 2 минуты

Отнести заявку к одному из типов:

1. **Готовая 3D-печать** — есть STL/STEP/3MF или понятная модель.
2. **Печать по фото/эскизу** — модели нет, нужно моделирование.
3. **Реверс-инжиниринг** — есть физическая деталь, нужна модель/аналог/посадка.
4. **Серия** — нужно несколько одинаковых деталей, важны сроки и повторяемость.
5. **Образовательный/ивент-проект** — мастер-класс, выставка, реквизит, демонстрация.

## 2. Что запросить, если данных мало

Минимальный безопасный ответ клиенту:

> Спасибо, задачу поняли. Чтобы посчитать точно, пришлите, пожалуйста: 2–4 фото детали/референса, один общий размер в мм, назначение детали и желаемое количество. Если есть файл STL/STEP/3MF — приложите его. После этого скажем технологию, срок и ориентир по стоимости.

## 3. Быстрый расчёт по категориям

| Тип | Что проверить | Ориентир ответа |
|---|---|---|
| Готовый файл | масштаб, толщина стенок, материал, нужна ли постобработка | печать + подготовка файла при необходимости |
| Фото/эскиз | хватает ли размеров, видны ли посадки, есть ли нагрузка | моделирование отдельно + печать |
| Реверс | критичные поверхности, точность, износ исходника | скан/обмер + CAD-модель + тестовая печать |
| Серия | количество, допуски, повторяемость, упаковка | сначала прототип, потом партия |
| Ивент/образование | дата, место, число участников, результат | программа/объект + подготовка материалов |

## 4. Когда не обещать сразу

Не обещать цену и срок без проверки, если:

- деталь несёт нагрузку или связана с безопасностью;
- нужна точная посадка, резьба, защёлка, герметичность;
- исходник только одно фото без размера;
- объект крупнее рабочей области принтера;
- требуется покраска, шлифовка, сборка или срочная доставка.

## 5. Внутренний следующий шаг

После каждой новой заявки сохранять в рабочую заметку:

- источник: сайт / Telegram / рекомендация;
- тип заявки;
- что уже есть: файл, фото, размеры, срок;
- что запросили;
- следующий контакт и дедлайн ответа.

Публиковать/отправлять клиенту только после подтверждения Никиты, если это не заранее разрешённый автоответ.
