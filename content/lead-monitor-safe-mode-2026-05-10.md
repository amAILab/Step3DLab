# Step3D — безопасный режим проверки заявок

Дата: 2026-05-10

## Что изменено

Для `scripts/check_step3d_leads.py` добавлен режим:

```bash
python3 scripts/check_step3d_leads.py --dry-run
```

Он нужен для проектных спринтов и ручных проверок: скрипт ищет новые заявки, но **не отправляет автоответы** и **не помечает новые письма как обработанные**.

## Поведение при сбое Google Workspace

Если `google-workspace`/`mcporter` недоступен, скрипт теперь завершает работу понятным статусом:

```text
LEAD_MONITOR_UNAVAILABLE: google-workspace/mcporter недоступен (exit N).
```

Это лучше, чем длинный traceback в cron-логах: сразу понятно, что проблема не в сайте и не в форме, а в доступности MCP Google Workspace.

## Обычный режим

Для боевого мониторинга остаётся прежняя команда:

```bash
python3 scripts/check_step3d_leads.py
```

Она может отправлять автоответы по новым заявкам, поэтому в задачах с ограничением «без внешних отправок» использовать только `--dry-run`.
